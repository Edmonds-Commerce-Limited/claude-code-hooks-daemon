# Release v3.0.1 - ClaudeMdInjector Content Loss Protection

**Release Date:** 2026-04-07
**Type:** Patch Release (Bug Fix)

## Summary

Version 3.0.1 adds defensive content-loss protection to the `ClaudeMdInjector`, which writes the `<hooksdaemon>...</hooksdaemon>` guidance section into project `CLAUDE.md` files on daemon startup. An environmental edge case could cause the injector to overwrite the entire file with only the auto-generated block, destroying all user content. This patch ensures that can never happen.

## Highlights

- **Content loss detection**: Before writing, the injector now extracts user content (everything outside the `<hooksdaemon>` tags) from both the original and proposed file. If user content would be lost, the write is aborted and an ERROR-level log is emitted.
- **Pre-inject backup**: A `.CLAUDE.md.pre-inject` backup is written before every modification to an existing `CLAUDE.md`, enabling manual recovery even if a future bug slips past the safety check.
- **Gitignore coverage**: The `.CLAUDE.md.pre-inject` backup file is now included in generated `.gitignore` entries (both fresh installs and upgrades) so it is never accidentally committed.
- **7 new unit tests** covering content loss detection, backup creation, user content extraction, and error logging.

## What Was Affected

Projects that had an existing `CLAUDE.md` with user content could lose that content when the daemon restarted (e.g. during an upgrade). The `<hooksdaemon>` block would remain but everything else — project rules, design principles, file indexes — would be silently deleted.

## Fixed

- `ClaudeMdInjector._is_user_content_preserved()` — new safety check comparing user content before and after replacement
- `ClaudeMdInjector._extract_user_content()` — extracts content outside `<hooksdaemon>` tags for comparison
- `ClaudeMdInjector._run_inject()` — aborts write on content loss, writes backup before modifying existing files
- `scripts/install/gitignore.sh` — includes `.CLAUDE.md.pre-inject` in generated gitignore entries

## Install / Upgrade

```bash
# New installation
git clone -b v3.0.1 https://github.com/Edmonds-Commerce-Limited/claude-code-hooks-daemon.git .claude/hooks-daemon
cd .claude/hooks-daemon && ./install.sh

# Upgrade from v3.0.0
cd .claude/hooks-daemon && git fetch && git checkout v3.0.1 && ./upgrade.sh
```
