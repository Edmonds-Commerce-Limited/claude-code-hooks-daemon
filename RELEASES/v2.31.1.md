# Release v2.31.1 - Stop Hook Branch 3 Race Condition Fix

**Release Date:** 2026-04-03
**Type:** Patch Release

## Summary

Version 2.31.1 fixes a race condition in the `auto_continue_stop` handler that caused Branch 3 (confirmation question auto-continue) to never fire. Analysis of 88 stop events showed zero Branch 3 triggers — every stop either matched Branch 2 (STOPPING BECAUSE:) or fell through to Branch 4 (force explanation). The root cause was that Branch 3 read from a stale transcript reader that was captured once at `handle()` entry, while Branch 2's retry logic reloaded the transcript internally without propagating the fresh reader back.

## Changes

### Fixed

- **`auto_continue_stop` Branch 3 stale reader race condition**: The confirmation-question auto-continue branch never fired because it used the original transcript reader captured at `handle()` entry, which could be stale after Branch 2's retry loop had reloaded the transcript. Fixed by reloading the transcript reader immediately before the Branch 3 check.

- **`auto_continue_stop` question evaluation guidance**: Added `get_claude_md()` guidance teaching Claude to distinguish tautological confirmation questions (rhetorical — just proceed) from genuine choice questions (require user input — use `STOPPING BECAUSE: need user input`). Prevents unnecessary stops on questions that don't need human decision-making.
