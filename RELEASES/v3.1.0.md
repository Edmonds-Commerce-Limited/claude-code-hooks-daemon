# Release v3.1.0 - Markdown Table Auto-Formatter

**Release Date:** 2026-04-10
**Type:** Minor Release (New Feature)

## Summary

Version 3.1.0 adds automatic markdown table formatting to the hooks daemon. A new PostToolUse handler (`markdown_table_formatter`) reformats markdown tables after every `Write` or `Edit` of a `.md` file, aligning table pipes vertically and normalising column widths using `mdformat` + `mdformat-gfm`. A companion CLI subcommand (`format-markdown`) enables ad-hoc batch formatting of existing files.

## Highlights

- **`markdown_table_formatter` PostToolUse handler** (priority 26, non-terminal): Automatically reformats markdown tables after Write/Edit of `.md` or `.markdown` files. Aligns pipes, widens delimiter rows, preserves `---` thematic breaks, keeps consecutive ordered-list numbering, and escapes asterisks in table cells per GFM spec. Never blocks — advisory only.

- **`format-markdown` CLI subcommand**: New command for ad-hoc markdown formatting:

  ```bash
  $PYTHON -m claude_code_hooks_daemon.daemon.cli format-markdown <path>       # Format file or directory
  $PYTHON -m claude_code_hooks_daemon.daemon.cli format-markdown --check <path> # CI mode — exit 1 if changes needed
  ```

- **`.CLAUDE.md.pre-inject` gitignore safety**: The installer script and `gitignore_safety_checker` SessionStart handler now ensure `.CLAUDE.md.pre-inject` is present in client project `.gitignore`, preventing the backup file from being accidentally committed.

## Added

- `markdown_table_formatter` PostToolUse handler with `get_claude_md()` guidance and `get_acceptance_tests()` definitions
- `format-markdown` CLI subcommand supporting single file, recursive directory, and `--check` mode
- `mdformat` (>=0.7) and `mdformat-gfm` (>=0.4) as runtime dependencies in `pyproject.toml`
- `.CLAUDE.md.pre-inject` pattern in `gitignore_safety_checker` required patterns
- `.CLAUDE.md.pre-inject` entry in `scripts/install/gitignore.sh` for fresh installs and upgrades
- 11 unit tests for `format-markdown` CLI subcommand
- Integration test coverage for handler response validation

## Fixed

- `scripts/install/gitignore.sh`: `ensure_root_gitignore` now checks and adds each entry independently instead of returning early when the daemon entry already exists
- `gitignore_safety_checker`: Fix-hint no longer appends trailing `/` to file patterns (was producing `.CLAUDE.md.pre-inject/`)

## Changed

- 257 project markdown files batch-formatted via `mdformat + mdformat-gfm` to establish consistent formatting baseline
- Handler count: 72 → 73 production handlers
