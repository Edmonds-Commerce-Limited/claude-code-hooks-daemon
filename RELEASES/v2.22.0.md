# Release v2.22.0 - Installer Enhancements: Plan Workflow & Handler Profiles

**Release Date:** 2026-03-13
**Type:** Minor Release

## Summary

Adds two new installer capabilities — plan workflow bootstrapping and handler profiles — plus fixes several installer bugs (prerequisite checks, daemon error surfacing, version module mismatch, and effort level error handling). These changes make the installation experience more robust and configurable, allowing new projects to immediately start with structured planning workflows and curated handler configurations.

## Highlights

- **Plan Workflow Bootstrap** (`PLAN_WORKFLOW=yes`): Installer can now create a complete `CLAUDE/Plan/` directory structure with templates for plan-based development.
- **Handler Profiles** (`HANDLER_PROFILE=recommended|strict`): Three tiers of handler configuration — minimal (default), recommended (16 handlers), or strict (24 handlers) — applied during install without manual config editing.
- **6 Installer Bugs Fixed**: Prerequisite validation, daemon error messages, version consistency, and error handling all improved.

## Changes

### Added

- **Plan Workflow Bootstrap** (`install.plan_workflow`): Creates `CLAUDE/Plan/` directory with README.md (plan index template), CLAUDE.md (lifecycle instructions), and Completed/ subdirectory. Idempotent — never overwrites existing files. Activated via `PLAN_WORKFLOW=yes` during installation (Step 14).

- **Handler Profiles** (`install.handler_profiles`): Three configuration tiers:
  - `minimal` (default): No extra handlers enabled
  - `recommended`: 16 quality/safety handlers (qa_suppression, tdd_enforcement, plan handlers, lint_on_edit, security_antipattern, etc.)
  - `strict`: recommended + 8 additional handlers (british_english, web_search_year, task_tdd_advisor, etc.)
  Applied via `HANDLER_PROFILE=recommended|strict` during installation (Step 15). Uses text-based YAML modification to preserve comments.

- **Installer Prerequisite Checks**: `check_python_importable` validates Python can actually import the daemon package after venv creation. Also checks for `uv` with automatic fallback to `pip`.

- **Daemon Error Surfacing**: `restart_daemon_verified` now captures and displays stderr from failed daemon restarts, showing the actual Python traceback instead of generic failure messages.

### Fixed

- **Version Module Mismatch**: `__init__.py` was defining `__version__` independently instead of importing from `version.py`, causing potential version drift. Now imports directly.

- **ModelContext Effort Error Handling**: `_read_effort_level()` now uses structured try/except with logging instead of bare exception suppression. OSError and JSON decode errors caught specifically and logged at warning level.

- **Status Line Effort Default Error Handling**: The error handling around reading effort level was using bare try/except which violated security standards. Now properly structured with specific exception types.

### Changed

- **Installer Steps 14-15**: Two new optional steps in `install_version.sh` using proper `if/then/else` blocks (SC2015 compliant).
- **Installer Completion Output**: Enhanced post-install summary with profile/plan status and customisation instructions.

## Upgrade Notes

No breaking changes. This is a backwards-compatible minor release.

**New installation options** (optional, existing installs unaffected):
```bash
# Enable plan workflow during install
PLAN_WORKFLOW=yes bash install.sh

# Apply recommended handler profile
HANDLER_PROFILE=recommended bash install.sh

# Both together
PLAN_WORKFLOW=yes HANDLER_PROFILE=strict bash install.sh
```

Existing installations can apply profiles manually by editing `.claude/hooks-daemon.yaml` handler `enabled:` flags.
