# Release v2.18.0 - LSP Enforcement Handler

**Release Date:** 2026-03-05
**Type:** Minor Release

## Summary

v2.18.0 adds the new LSP Enforcement Handler, which steers LLMs toward semantic LSP tools instead of Grep/Bash grep for symbol lookups. Also fixes the status line upgrade indicator to show both current and target versions.

## Changes

### Added

- **LspEnforcementHandler** (PreToolUse, priority 38): Detects Grep and Bash(grep/rg) patterns that look like symbol lookups (class/def/function/interface definitions, PascalCase identifiers, snake_case identifiers, import statements) and steers the LLM toward LSP tools (goToDefinition, findReferences, workspaceSymbol) instead of slow text searches. Configurable modes:
  - `block_once` (default): Deny first grep-that-looks-like-LSP, allow retries
  - `advisory`: Always allow with LSP guidance in system-reminder
  - `strict`: Always deny symbol-like grep patterns
  - `no_lsp_mode` option (`block`/`advisory`/`disable`): Controls behaviour when LSP is not configured via `ENABLE_LSP_TOOL` env var

### Fixed

- **Status Line Upgrade Indicator**: Fixed the upgrade-available indicator showing only the target version. Now displays both current and target version (e.g. `v2.17.3 → v2.18.0`). Falls back to `upgrade → vX.Y.Z` if current version unavailable.
- **NpmCommandHandler False Positive on Logical OR**: Fixed pipe detection regex matching `||` (logical OR) as a pipe operator, causing false blocks on commands like `npx hooks-daemon restart || fallback`. Now correctly distinguishes single `|` (pipe) from `||` (logical OR).
- **PHP TDD Strategy Interface Skipping**: Fixed TDD enforcement handler requiring test files for PHP interface files. Added filename convention (`*Interface.php`) and content inspection detection methods. Extended `should_skip()` protocol to accept optional `content` parameter across all 11 language strategies.
- **Silent Exception Hiding in TDD Handler**: Removed `try/except` blocks that silently swallowed errors in TDD handler path mapping. Callers already guard inputs, so these exceptions cannot occur — removing them enables fail-fast behaviour.
- **Skill Tool Invocation Clarification**: Fixed CLAUDE.md install templates showing `/hooks-daemon restart` in bash code blocks, causing Claude to run it as a bash command instead of using the Skill tool. Changed `disable-model-invocation` to `false` so Claude can invoke the skill.

## Upgrade Instructions

No breaking changes. Standard upgrade procedure applies.

```bash
cd .claude/hooks-daemon
git fetch --tags
git checkout v2.18.0
untracked/venv/bin/pip install -e .
```

Or using the upgrade skill:

```bash
/hooks-daemon upgrade v2.18.0
```

After upgrading, restart the daemon:

```bash
untracked/venv/bin/python -m claude_code_hooks_daemon.daemon.cli restart
```

### Enabling the LSP Enforcement Handler

The handler is disabled by default. To enable it, add to your `hooks-daemon.yaml`:

```yaml
handlers:
  pre_tool_use:
    lsp_enforcement:
      enabled: true
      priority: 38
      options:
        mode: block_once       # advisory | block_once | strict
        no_lsp_mode: block     # block | advisory | disable
```
