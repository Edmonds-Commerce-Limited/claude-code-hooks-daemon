# Release v2.19.0 - TDD Collocated Test Support & Go Lint Fix

**Release Date:** 2026-03-06
**Type:** Minor Release

## Summary

Version 2.19.0 adds collocated test file support to the TDD enforcement handler and fixes Go lint commands to use correct module root directories and package-level vetting. These changes eliminate false blocking for Go, React/Jest, and other projects that use collocated test conventions.

## Highlights

- **TDD Collocated Test Support**: Three test location modes (`separate`, `collocated`, `test_subdir`) all enabled by default
- **Go Lint Module Root Fix**: Go lint tools now find `go.mod` and vet package directories instead of single files
- **6568 Total Tests**: Comprehensive test coverage maintained at 95.0%

## Changes

### Added - TDD Collocated Test Support (Plan 00076, commit c882817)

The TDD enforcement handler now supports collocated test file layouts in addition to the traditional separate `tests/` directory.

**Three Test Location Modes**:

- **`separate`**: Traditional `tests/` directory tree mirroring `src/` (existing behavior)
- **`collocated`**: Test file next to source file (e.g., Go `_test.go`, React/Jest `*.test.ts`)
- **`test_subdir`**: `__tests__/` subdirectory alongside source files (Jest/Vitest convention)

All three modes are enabled by default, making the handler work out-of-the-box for Go, JavaScript, TypeScript, and any other language using collocated test conventions.

**New Configuration Option**:

```yaml
handlers:
  pre_tool_use:
    tdd_enforcement:
      enabled: true
      test_locations: [separate, collocated, test_subdir]  # default: all three
```

**Implementation Details**:

- New constants: `_TEST_LOCATION_SEPARATE`, `_TEST_LOCATION_COLLOCATED`, `_TEST_LOCATION_TEST_SUBDIR`, `_TEST_SUBDIR_NAME`, `_DEFAULT_TEST_LOCATIONS`
- New `_test_locations` config attribute with `_effective_test_locations` property
- New static methods: `_map_collocated_test_path()`, `_map_test_subdir_path()`
- Modified `_get_test_file_paths()` to gate existing logic and append new candidates
- 27 new tests covering constants, config, path methods, integration, and regressions

### Fixed - Go Lint Module Root and Package Vetting (commit 16d3d8b)

Go lint commands (`go vet`, `golangci-lint`) now run from the correct module root directory and vet entire package directories instead of single files.

**Problem**: Go lint tools require running from within the Go module tree (where `go.mod` lives). The handler was running from the daemon's working directory, causing "no such file or directory" errors for `go.mod`. Additionally, single-file vetting (`go vet file.go`) couldn't resolve types defined in other files within the same package.

**Fix**:

- Added `_find_module_root()` static method that walks up from the file's directory to find marker files (e.g., `go.mod`)
- Added `_MODULE_ROOT_MARKERS` dict for extensible language support
- Converts Go file paths to module-relative package directory paths (e.g., `./internal/github/`)
- Passes module root as `cwd` to `subprocess.run()`
- 5 new tests covering module root detection and Go-specific behavior

## Installation

### New Installation

```bash
git clone -b v2.19.0 https://github.com/cravend/claude-code-hooks-daemon.git .claude/hooks-daemon
cd .claude/hooks-daemon && ./install.sh
```

### Upgrade from v2.18.0

```bash
cd .claude/hooks-daemon
git fetch origin
git checkout v2.19.0
./install.sh
```

No configuration changes required. The new `test_locations` option uses sensible defaults (all three modes enabled).
