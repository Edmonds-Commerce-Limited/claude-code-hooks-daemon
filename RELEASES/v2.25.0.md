# Release v2.25.0 - Context Window Size Threshold Tiers

**Release Date:** 2026-03-19
**Type:** Minor Release

## Summary

Version 2.25.0 introduces context-window-size-based threshold tiers for the `model_context` status line handler. The context percentage colour coding is now keyed by the actual context window size (200k vs 1M tokens) rather than model name, providing tighter thresholds for large-context models where even moderate usage percentages represent enormous token payloads.

## Highlights

- **Window-size-based context thresholds**: Models with 1M context windows now show orange at 30% and red at 40%, because 300k-400k tokens is already an enormous context to push back and forth per API call. Standard 200k models keep the existing thresholds (orange at 51%, red at 76%).
- **Fully configurable per tier**: All thresholds are configurable via handler options in `.claude/hooks-daemon.yaml` using the pattern `200k_orange_pct`, `1000k_orange_pct`, etc.
- **Future-proof design**: Adding a new tier (e.g. 2000k) requires only two new config options and one additional condition in `_resolve_tier_thresholds`. Unknown window sizes fall back to the largest configured tier.

## Changes

### Added

#### Context Window Size Threshold Tiers for `model_context` Status Handler

The status line context percentage indicator now uses tiers keyed by `context_window_size` from the hook event data. This replaces the previous flat threshold approach with a tiered system:

- **200k tier** (standard models — Sonnet, Haiku, Opus-200k): Green 0-25%, Yellow 26-50%, Orange 51-75%, Red 76%+
- **1000k tier** (Opus-1M and future large-context models): Green 0-14%, Yellow 15-29%, Orange 30-39%, Red 40%+

The tighter 1M thresholds exist because at that scale, 30% is already 300k tokens and 40% is 400k tokens — quality degrades from noise, latency spikes, and costs balloon well before the window is technically full.

**Configuration** (in `.claude/hooks-daemon.yaml`):

```yaml
model_context:
  enabled: true
  priority: 10
  options:
    200k_orange_pct: 51    # default: 51
    200k_red_pct: 76       # default: 76
    1000k_orange_pct: 30   # default: 30
    1000k_red_pct: 40      # default: 40
```

**Files changed**: `src/claude_code_hooks_daemon/handlers/status_line/model_context.py`, `tests/unit/handlers/status_line/test_model_context.py`, `.claude/hooks-daemon.yaml`

### Changed

#### Context Thresholds Keyed by Window Size, Not Model Name

Replaces the previous approach of checking the model display name for "Opus" with a model-agnostic window-size lookup. Any model (current or future) with a 1M context window automatically receives tighter thresholds; any model at 200k uses standard thresholds. This makes the feature robust to new model releases without code changes.

### Fixed

#### Error Hiding Exclusion Line Number Updated

The `try/except` block in `_read_effort_level` shifted from line 165 to line 215 after adding context-window-size tier constants and configurable threshold fields. The error-hiding exclusion in `error_hiding_exclusions.json` was updated to match.

**Files changed**: `scripts/qa/error_hiding_exclusions.json`
