# Release v2.21.1 - Statusline Config Sync

**Release Date:** 2026-03-12
**Type:** Patch Release

## Summary

Fixes the statusline effort level default to match Claude Code's actual default (medium, not high), and adds auto-configuration of `effortLevel` and `alwaysThinkingEnabled` in `~/.claude/settings.json` on SessionStart so the statusline always reflects explicit configuration.

## Changes

### Fixed

- **Statusline Effort Default Corrected**: `ModelContextHandler` was defaulting to "high" effort bars when `effortLevel` was absent from `~/.claude/settings.json`, but Claude Code actually defaults to "medium". The `_EFFORT_DEFAULT` constant is now `"medium"`, so the status line correctly reflects Claude Code's behaviour when no explicit effort level is configured.

- **Auto-Set Effort Config on SessionStart**: `OptimalConfigCheckerHandler` now writes `effortLevel=high` and `alwaysThinkingEnabled=true` to `~/.claude/settings.json` at the start of each new session when those keys are absent. This keeps the status line in sync with the desired optimal configuration. The write is safe: reads the file first and aborts if the read fails, preventing clobbering of existing settings.

### Changed

- **Release Documentation**: Added exact `gh release view` JSON query command to post-release verification (Step 11) in `CLAUDE/development/RELEASING.md`.
