# Release v2.31.0 - Fresh Clone Experience and Command Redirection Fixes

**Release Date:** 2026-04-03
**Type:** Minor Release

## Summary

Version 2.31.0 adds a new `/hooks-daemon install` skill for fresh clone environments and fixes several issues with the command redirection system. The install skill fills a critical gap where freshly cloned projects with hooks daemon configuration had no automated way to install the daemon — hook scripts now direct the LLM to use the skill instead of referencing non-existent documentation files. Command redirection output is moved from the daemon's runtime directory to `/tmp/` to prevent CLAUDE.md cascade reads, and redirection is now disabled by default across all handlers.

## Highlights

- **New `/hooks-daemon install` skill** — freshly cloned projects can now install the daemon via a deployed skill instead of manual steps
- **Command redirection writes to `/tmp/hooks-daemon-cmd/`** — output files no longer trigger CLAUDE.md cascade loading
- **Command redirection disabled by default** — `pipe_blocker`, `gh_issue_comments`, and `npm_command` handlers now default to blocking-only mode (can be re-enabled via config)
- **Placeholder URLs fixed** — all `your-org` references in skill files replaced with `Edmonds-Commerce-Limited`
- **Handler ABC version registry accuracy** — `_ABSTRACT_METHOD_VERSIONS` now correctly reflects `get_claude_md()` version
- **`validate-project-handlers` exit code fix** — CLI now returns exit 1 on load failures for CI integration

## Changes

### Added

- **Handler ABC version registry accuracy**: Updated `_ABSTRACT_METHOD_VERSIONS` registry in `project_loader.py` to ensure the `get_claude_md()` method version entry is accurate, preventing false positive version mismatch errors when validating project handlers.

- **Release process checklist for Handler ABC methods**: Added a detection step to the release process for new `@abstractmethod` entries on the `Handler` base class — ensuring upgrade guides document the method name, version added, and the required stub for project handlers.

- **Upgrade guide v2.29 to v2.30**: Created comprehensive upgrade guide in `CLAUDE/UPGRADES/v2/v2.29-to-v2.30/` documenting the `get_claude_md()` breaking change introduced in v2.30.0, including before/after config examples and migration steps.

### Fixed

- **`validate-project-handlers` now returns exit 1 on load failures**: Previously, the CLI command reported handler load failures but exited with code 0, making it unusable in CI scripts for detecting problems. Now correctly exits with code 1 when any project handler fails to load.

- **Quote-aware pipe splitting in pipe blocker handler**: The pipe blocker previously split commands on `|` characters inside quoted strings, causing false positives on legitimate commands that contained quoted pipe characters. Now correctly ignores `|` inside single and double quotes.

- **Project handler load failures no longer crash the daemon**: A single malformed or incompatible project handler previously caused the entire daemon startup to fail. Now failed project handlers are skipped with an error log entry, allowing the daemon to start with the remaining valid handlers.

- **Command redirection output to `/tmp/` instead of daemon dir**: The async command redirection feature (pipe blocker, gh issue comments, npm command) previously wrote output files into the daemon's runtime directory. When the LLM read those files, it triggered a CLAUDE.md cascade loading all hooks-daemon documentation. Changed to write to `/tmp/hooks-daemon-cmd/` with automatic cleanup of files older than 1 hour.

- **Fresh clone install experience**: Hook scripts on freshly cloned projects now direct the LLM to use `/hooks-daemon install` instead of referencing the non-existent `CLAUDE/LLM-INSTALL.md` file. New install skill and shell script added to the deployed skills package.

- **Placeholder URLs replaced**: Fixed `your-org/hooks-daemon` and `your-org/claude-code-hooks-daemon` references across skill files to use the correct `Edmonds-Commerce-Limited/claude-code-hooks-daemon` GitHub URL. Also fixed `curl | bash` patterns to use the safer download-then-inspect approach.

- **Command redirection disabled by default**: All three handlers with command redirection (`pipe_blocker`, `gh_issue_comments`, `npm_command`) now default to `command_redirection: false` to avoid the CLAUDE.md cascade issue. Can be re-enabled via handler options in `.claude/hooks-daemon.yaml`.

### Changed

- **RELEASING.md condensed from 1,178 to 210 lines**: Refactored the release documentation to be significantly more concise while preserving all essential steps, making it faster to read and easier to follow during releases.
