# Release v2.26.0 - Async Command Redirection & Plan Numbering Fix

**Release Date:** 2026-03-23
**Type:** Minor Release

## Summary

Version 2.26.0 introduces asynchronous command redirection for blocking handlers, preventing hook timeouts when redirected commands are slow (e.g. `pytest`, `npm test`). It also fixes a plan numbering bug where date-formatted directories in legacy archives inflated the sequential counter.

## Highlights

- **Async command redirection (`launch_and_save`)**: The pipe blocker handler now spawns redirected commands as detached background processes instead of waiting synchronously. This prevents hook timeouts when the base command is slow. A daemon reaper thread prevents zombie process accumulation.
- **Plan numbering regex fix**: Date-formatted directories like `2026-01-12` in legacy archives no longer inflate the plan counter. The regex now requires a letter after the hyphen, correctly distinguishing plan directories from date directories.
- **Install/upgrade docs overhaul**: Handler enablement is now strongly emphasised in both `LLM-INSTALL.md` and `LLM-UPDATE.md`, preventing agents from leaving most handlers disabled after installation.

## Changes

### Added

#### Async Command Redirection (`launch_and_save`) for Pipe Blocker

The pipe blocker's command redirection now uses a new non-blocking `launch_and_save()` function instead of the synchronous `execute_and_save()`. When blocking a command piped to `tail`/`head`, the handler immediately spawns the base command as a detached background process (`start_new_session=True`) and returns the deny message without waiting for completion.

Key design details:
- Background process output is appended to a file via a constant bash wrapper script
- Command arguments are passed positionally via `$@` — no shell interpolation of user-supplied values
- A daemon reaper thread calls `wait()` on the spawned process to prevent zombie accumulation
- The reaper runs as a daemon thread so it won't block daemon shutdown

**Files changed**: `src/claude_code_hooks_daemon/core/command_redirection.py`, `src/claude_code_hooks_daemon/handlers/pre_tool_use/pipe_blocker.py`, `tests/unit/core/test_command_redirection.py`, `tests/unit/handlers/pre_tool_use/test_pipe_blocker.py`

### Fixed

#### Plan Numbering Regex Matches Date Directories

The regex `^(\d+)-` in `plan_numbering.py` and `validate_plan_number.py` matched date-formatted directories such as `2026-01-12` in legacy archives, extracting `2026` as a plan number and causing the sequential counter to jump from ~32 to ~2027. Fixed by tightening the regex to `^(\d{1,5})-[a-zA-Z]`, which requires a letter immediately after the hyphen — excluding date patterns (digit after hyphen) while correctly matching valid plan directories like `00032-svc-feature`.

**Files changed**: `src/claude_code_hooks_daemon/handlers/utils/plan_numbering.py`, `src/claude_code_hooks_daemon/handlers/pre_tool_use/validate_plan_number.py`, `tests/unit/handlers/utils/test_plan_numbering.py`, `tests/unit/handlers/pre_tool_use/test_validate_plan_number.py`

#### Install and Upgrade Docs Emphasise Handler Enablement

`CLAUDE/LLM-INSTALL.md` and `CLAUDE/LLM-UPDATE.md` previously treated handler configuration as optional, causing agents to leave most handlers disabled after install. Renamed "Configure Handlers (Optional)" to "Enable and Configure Handlers (CRITICAL)", added a full handler category table with enablement recommendations, added a comprehensive example config enabling 20+ handlers, and made the Handler Status Report section mandatory with a 30+ enabled handler target.

**Files changed**: `CLAUDE/LLM-INSTALL.md`, `CLAUDE/LLM-UPDATE.md`

#### QA Violations in Async Command Redirection

Resolved error-hiding audit violations introduced by the `launch_and_save` implementation: converted silent `pass` in reaper thread to `logger.debug`, updated stale line-number references in `error_hiding_exclusions.json`, and added new exclusion entry for the reaper thread's intentional catch-all.

**Files changed**: `src/claude_code_hooks_daemon/core/command_redirection.py`, `scripts/qa/error_hiding_exclusions.json`, `tests/unit/core/test_command_redirection.py`
