# Release v2.21.0 - Pseudo-Events, Nitpick Handlers & PlanWorkflowConfig

**Release Date:** 2026-03-11
**Type:** Minor Release

## Summary

Version 2.21.0 introduces the pseudo-event infrastructure -- a system for synthetic daemon-generated events that piggyback on real events with frequency control. The first concrete pseudo-event ("nitpick") periodically audits conversation transcripts for dismissive and hedging language patterns. This release also adds a top-level `plan_workflow` config block, the `generate-docs` CLI command, dismissive language detection at Stop events, and several bug fixes including an xargs bypass of the inplace-edit blocker.

## Highlights

- **Pseudo-Event Infrastructure**: New `PseudoEventDispatcher` for synthetic events with `N/D` frequency notation, per-session counters, shared setup functions, and result merging
- **Nitpick Pseudo-Event**: Periodic transcript auditing (every 5 PreToolUse events) with dismissive and hedging language handlers
- **DismissiveLanguageDetectorHandler**: Stop event advisory with 30+ compiled regexes across 4 pattern categories
- **generate-docs CLI Command**: Auto-generates `.claude/HOOKS-DAEMON.md` from live config and handler metadata
- **PlanWorkflowConfig**: Centralised plan configuration with backwards-compatible migration from handler options
- **Plan Redirect Improvements**: ALLOW flow for project plan directory writes + 5-digit plan folder support
- **7115+ Total Tests**: Comprehensive test coverage maintained at 95%+

## Changes

### Added - Pseudo-Event Infrastructure (Plan 00082)

New core infrastructure for synthetic daemon-generated events:

- **PseudoEventDispatcher**: Parses frequency notation (`event_type:N/D`), maintains per-session counters, runs shared setup functions, and dispatches through dedicated handler chains
- **Nitpick Pseudo-Event**: First concrete pseudo-event triggered every 5 PreToolUse events by default. Reads conversation transcript incrementally using byte-offset tracking
- **NitpickDismissiveLanguageHandler**: Audits assistant messages for dismissive language patterns. Reuses pattern lists from DismissiveLanguageDetectorHandler as single source of truth
- **NitpickHedgingLanguageHandler**: Audits assistant messages for hedging language. Reuses pattern lists from HedgingLanguageDetectorHandler

### Added - DismissiveLanguageDetectorHandler (Plan 00079)

New Stop event advisory handler that scans the last assistant message for dismissive language patterns:
- "Not our problem" deflection
- "Out of scope" avoidance
- "Someone else's job" pushing
- Defer/ignore phrases

Four pattern categories with 30+ compiled regexes. Priority 58 (advisory range).

### Added - generate-docs CLI Command (Plan 00080)

New `DocsGenerator` class and `generate-docs` CLI command that generates `.claude/HOOKS-DAEMON.md` from live config and handler metadata. Produces an accurate, auto-generated summary of active handlers (including project handlers and plugins), plan mode settings, and config reference. Integrated into the installer.

### Added - PlanWorkflowConfig (Plan 00086)

New `plan_workflow` top-level configuration block centralising plan-related settings:

```yaml
plan_workflow:
  enabled: true
  directory: "CLAUDE/Plan"
  workflow_docs: "CLAUDE/PlanWorkflow.md"
  enforce_claude_code_sync: false
```

Automatic migration reads `track_plans_in_project` from `markdown_organization` handler options for backwards compatibility.

### Added - Plan Redirect Improvements (Plan 00086)

- Plan redirect handler now supports an ALLOW flow for writes targeting the project plan directory
- Recognises 5-digit plan folder numbers (e.g. `00085`, `00086`) alongside existing 3-digit format
- `plansDirectory` sync enforcement when `plan_workflow.enforce_claude_code_sync` is enabled

### Added - TranscriptReader Incremental Reading

`TranscriptReader` now supports incremental JSONL reading via `read_incremental(path, byte_offset)`, returning only messages added since the last read position. Also adds `UUID` field to parsed messages and `filter_assistant_messages()` static method.

### Added - DestructiveGitHandler: Amend Flag Blocking

Added pattern 9 to the destructive git handler, blocking all `git commit` with the `--amend` flag. Amending rewrites the previous commit and creates messy history — new commits are always preferred. Includes acceptance test for playbook generation.

### Added - PostClearAutoExecuteHandler (Prototype)

UserPromptSubmit advisory handler that detects the first prompt of a new session and injects guidance for the agent to execute the instruction immediately. Intended to address the `/clear <instruction>` idle agent pattern. Marked as prototype; the core problem (Claude Code's `local-command-caveat` preventing auto-execution) cannot be fully solved via hooks (see Plan 00087).

### Changed

- **DaemonController Pseudo-Event Integration**: `DaemonController.initialise()` now accepts `pseudo_events_config` and `plan_workflow` parameters with pseudo-event dispatch wired into the event loop
- **PlanWorkflowConfig Migration**: Config model validator automatically migrates handler-level plan options to the new top-level block
- **HandlerRegistry receives plan_workflow**: Plan-related handlers read configuration from the centralised config

### Fixed

- **Block xargs Bypass of Inplace-Edit Blocker** (Plan 00084): Now matches `xargs sed`, `xargs perl -pi`, and `xargs awk -i` patterns
- **validate_plan_number Hardcoded Plan Directory** (Plan 00083): Handler now reads the configured plan directory from `plan_workflow.directory`
- **TranscriptReader Crash on Real Transcripts**: More defensive parsing against missing or malformed content blocks
- **Dismissive Language Pattern Gaps**: Extended pattern set to cover additional deflection phrases
- **Version Cache Flush After Upgrade**: Cache flushed when new version detected, ensuring correct status line display
- **Error Hiding Exclusions Drift**: Updated exclusion line numbers to match current source
- **QA Violations in DocsGenerator and CLI**: Fixed ruff lint and mypy type errors
