# Release v2.23.0 - Command Redirection & Handler Fixes

**Release Date:** 2026-03-15
**Type:** Minor Release

## Summary

Introduces command redirection — a mechanism that allows blocking handlers to automatically execute corrected commands and return the output to Claude in a single turn. Previously, when a handler blocked a command and suggested a fix, the agent had to manually re-run the corrected command, wasting a turn. Now three handlers (`pipe_blocker`, `npm_command`, `gh_issue_comments`) can optionally execute the corrected version and save the output for immediate inspection. Also fixes two bugs: `auto_approve_reads` was entirely non-functional due to matching on a non-existent field, and `markdown_organization` was creating duplicate plan entries.

## Highlights

- **Command Redirection Core**: New utility module enabling blocking handlers to execute corrected commands automatically, eliminating wasted turns
- **3 Handlers Enhanced**: `pipe_blocker`, `npm_command`, and `gh_issue_comments` now support automatic command redirection
- **`ask_user_question_blocker` Handler**: New handler (disabled by default) for fully unattended autonomous workflows
- **`auto_approve_reads` Bug Fix**: Handler was dead code — now actually functional
- **`markdown_organization` Bug Fix**: Plan folder creation no longer produces duplicate entries

## Changes

### Added

- **Command Redirection Core Utility** (`utils.command_redirection`): Reusable mechanism for blocking handlers to automatically execute corrected commands and save output to files. Eliminates wasted turns when a handler blocks a command and suggests a corrected version — Claude receives the educational deny message and the command result in a single turn. Includes `CommandRedirectionResult` dataclass, `execute_and_save()`, `format_redirection_context()`, and `cleanup_old_files()`. 24 unit tests.

- **Command Redirection for `gh_issue_comments` Handler**: When blocking a `gh issue view` command that lacks `--comments`, the handler now automatically executes the corrected command (with `--comments`) and saves the output to a file. Configurable via `command_redirection: true/false` option (default: enabled). Graceful fallback to block-only if execution fails.

- **Command Redirection for `npm_command` Handler**: When blocking raw `npm`/`npx` commands, the handler now automatically executes the corrected `llm:`-prefixed command and saves the output. Configurable via `command_redirection` option (default: enabled). Graceful fallback to block-only if execution fails.

- **Command Redirection for `pipe_blocker` Handler**: When blocking commands piped to `tail`/`head`, the handler now automatically executes the base command (without the pipe) and saves the output to a file. Configurable via `command_redirection` option (default: enabled). Graceful fallback to block-only if execution fails.

- **`ask_user_question_blocker` Handler**: New PreToolUse handler (disabled by default) that blocks `AskUserQuestion` tool calls for fully unattended autonomous workflows. Useful in CI or scheduled agent runs where interactive prompts are not acceptable.

- **Transcript-Based Debugging Guide**: Added `CLAUDE/DEBUGGING_TRANSCRIPTS.md` documenting the technique of using conversation transcripts for post-mortem debugging of hook event issues — analysing actual event payloads from real sessions to investigate handler failures.

### Fixed

- **`auto_approve_reads` Handler Dead Code Bug**: Handler was matching on `permission_type` field which does not exist in real `PermissionRequest` events, making it entirely non-functional. Rewrote to match on `tool_name` (`Read`, `Glob`, `Grep`) which is the actual field present in real events. Handler is now functional.

- **`markdown_organization` Plan Folder Write Returns DENY**: `_handle_plan_write()` previously returned `Decision.ALLOW` after creating the numbered plan folder, causing both a flat file and a folder to exist simultaneously. Now returns `Decision.DENY` with a descriptive reason explaining the redirect path and instructing the agent to rename the folder to something semantic.

### Changed

- **Handler Reference Documentation**: Added `command_redirection` option documentation to `docs/guides/HANDLER_REFERENCE.md` covering all three handlers that support it.
- **Config Examples Updated**: `.claude/hooks-daemon.yaml` and `.claude/hooks-daemon.yaml.example` updated with `command_redirection` option examples.

## Upgrade Notes

No breaking changes. This is a backwards-compatible minor release.

**New handler options** (optional, existing installs unaffected):

```yaml
# Enable command redirection (already enabled by default for these handlers)
handlers:
  pre_tool_use:
    pipe_blocker:
      options:
        command_redirection: true
    npm_command:
      options:
        command_redirection: true
    gh_issue_comments:
      options:
        command_redirection: true

    # Enable for unattended workflows (disabled by default)
    ask_user_question_blocker:
      enabled: true
```
