# Release v3.2.0 - Universal Continuation Instruction for Blocked Tools

**Release Date:** 2026-04-10
**Type:** Minor Release (Feature)

## Summary

Version 3.2.0 addresses a common agent behaviour problem: when a PreToolUse hook blocks a tool call, agents sometimes interpret the block as a signal to stop working entirely, rather than adjusting their approach and continuing. This release adds a universal continuation instruction to all PreToolUse deny responses and reinforces it in the auto-generated CLAUDE.md guidance.

## Highlights

- **Universal deny continuation suffix**: All PreToolUse DENY responses now automatically append "Do not stop working. Modify your approach using the guidance above and continue." to the block reason. This is a single-point change in `HookResult._format_pre_tool_use_response()` that covers all blocking handlers without requiring individual handler modifications.

- **CLAUDE.md hooksdaemon section guidance**: The auto-generated `<hooksdaemon>` section intro now includes a universal note: "When a tool is blocked by a handler, do not stop working. Read the block reason, modify your approach, and continue with your task."

## Added

- Universal continuation instruction appended to all PreToolUse deny responses via `_DENY_CONTINUATION_SUFFIX` constant in `hook_result.py`
- Continuation guidance in the `<hooksdaemon>` section intro injected into project CLAUDE.md files
