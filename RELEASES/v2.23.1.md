# Release v2.23.1 - Git Filemode Protection and Auto-Continue Improvements

**Release Date:** 2026-03-16
**Type:** Patch Release

## Summary

Version 2.23.1 addresses silent hook permission loss caused by `git core.fileMode=false` and improves autonomous session continuity. A new SessionStart handler warns when filemode tracking is disabled, and the install/upgrade process now force-commits hook scripts as executable via `git update-index --chmod=+x`. The `auto_continue_stop` handler gains a `continue_on_errors` option for uninterrupted error recovery.

## Highlights

- **Git Filemode Protection**: New `GitFilemodeCheckerHandler` warns on session start when `core.fileMode=false` risks breaking hook permissions
- **Force-Executable Install Step**: `git update-index --chmod=+x` ensures hooks are committed as `100755` regardless of filemode setting
- **Auto-Continue on Errors**: New `continue_on_errors` option (default: `true`) lets sessions recover from Bash errors without blocking

## Changes

### Added

- **`GitFilemodeCheckerHandler`** (SessionStart, priority 53, advisory): Detects `git core.fileMode=false` and warns about hook scripts potentially losing executable permissions after git operations (checkout, merge, rebase). Runs on new sessions only (not resumes). Recommends `git config core.fileMode true`. 21 unit tests with full coverage.

- **`git_force_executable` install step**: New function in `scripts/install/hooks_deploy.sh` that runs `git update-index --chmod=+x` on all tracked hook scripts during both install and upgrade. Ensures hooks are stored as `100755` in git's index regardless of `core.fileMode` setting, preventing the root cause of silent permission loss.

- **`continue_on_errors` option for `auto_continue_stop`**: New boolean option (default: `true`) that controls whether the handler auto-continues when Claude's stop message contains error patterns (`"error:"`, `"failed:"`). When enabled, sessions no longer block waiting for user intervention after Bash failures.

### Changed

- **`auto_continue_stop` advisory reason improved**: Reason message now guides toward truly stuck scenarios (blocked on user input, ambiguous requirements, missing credentials) rather than vague continuation prompts.

- **Handler reference documentation**: Updated with `continue_on_errors` option entry for `auto_continue_stop`.

## Upgrade Instructions

Standard upgrade — no breaking changes:

```bash
cd .claude/hooks-daemon
git fetch origin
git checkout v2.23.1
./scripts/install/upgrade.sh
```

The new `git_force_executable` step runs automatically during upgrade, fixing any existing permission issues.

### New Handler Configuration

The `git_filemode_checker` handler is enabled by default. To customise:

```yaml
# .claude/hooks-daemon.yaml
handlers:
  session_start:
    git_filemode_checker:
      enabled: true   # Strongly recommended
      priority: 53
```

### Auto-Continue Configuration

To disable auto-continue on errors (revert to pre-v2.23.1 behaviour):

```yaml
handlers:
  stop:
    auto_continue_stop:
      enabled: true
      priority: 10
      options:
        continue_on_errors: false  # Block on error patterns
```
