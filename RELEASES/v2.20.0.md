# Release v2.20.0 - SecurityAntipatternHandler & 12 Language Strategies

**Release Date:** 2026-03-09
**Type:** Minor Release

## Summary

Version 2.20.0 introduces the SecurityAntipatternHandler, a comprehensive security analysis handler that blocks hardcoded secrets (OWASP A02) and code injection patterns (OWASP A03) across 12 language strategies using the Strategy Pattern. This release also adds structured transcript parsing with ContentBlock, shared stop hook utilities, progressive pipe blocker verbosity, and fixes AutoContinueStopHandler overriding AskUserQuestion.

## Highlights

- **SecurityAntipatternHandler**: Blocks secrets and injection patterns across 12 language strategies (Secrets, Python, JS/TS, PHP, Go, Ruby, Java, Kotlin, C#, Rust, Swift, Dart)
- **Strategy Pattern Architecture**: Each language is an independent strategy class with SecurityStrategy Protocol and SecurityStrategyRegistry
- **ContentBlock Dataclass**: Structured transcript parsing with typed tool_use, text, and tool_result blocks
- **Pipe Blocker Progressive Verbosity**: First block is verbose, subsequent blocks are terse
- **AutoContinueStopHandler Fix**: No longer overrides AskUserQuestion tool use
- **6750+ Total Tests**: Comprehensive test coverage maintained at 95%+

## Changes

### Added - SecurityAntipatternHandler (Plan 00078)

New PreToolUse handler that blocks hardcoded secrets and code injection patterns using the Strategy Pattern with 12 language strategies.

**Architecture**:
- `SecurityStrategy` Protocol defines the interface for language-specific security checks
- `SecurityPattern` frozen dataclass holds pattern metadata (name, regex, severity, description)
- `SecurityStrategyRegistry` manages strategy registration and lookup by file extension
- Each language is an independent strategy class, following Open/Closed Principle

**Language Strategies**:

| Language | Patterns | Examples |
|----------|----------|---------|
| Secrets (universal) | API keys, tokens, passwords, private keys, connection strings | `sk_live_`, `AKIA`, `password = "..."` |
| Python | 7 patterns | `eval()`, `exec()`, `os.system()`, `subprocess` with `shell=True`, `pickle.load()` |
| JavaScript/TypeScript | 5 patterns | `eval()`, `innerHTML`, `document.write()`, `Function()`, `child_process.exec()` |
| PHP | 7 patterns | `eval()`, `exec()`, `system()`, `shell_exec()`, `passthru()`, `unserialize()` |
| Go | 3 patterns | `template.HTML()`, `template.JS()`, `template.URL()` |
| Ruby | 7 patterns | `eval()`, `system()`, `exec()`, `instance_eval()`, `Marshal.load()`, `IO.popen()` |
| Java | 4 patterns | `Runtime.getRuntime().exec()`, `ObjectInputStream`, `XMLDecoder` |
| Kotlin | 4 patterns | `Runtime.getRuntime().exec()`, `ObjectInputStream()`, `XMLDecoder()` |
| C# | 4 patterns | `Process.Start()`, `BinaryFormatter`, `LosFormatter` |
| Rust | 2 patterns | `from_raw_parts()`, `transmute()` |
| Swift | 3 patterns | `Process()`, `evaluateJavaScript()`, `NSKeyedUnarchiver.unarchiveObject()` |
| Dart | 3 patterns | `Process.run()`, `Process.start()`, `innerHTML` assignment |

### Added - ContentBlock Dataclass & TranscriptReader Query Methods

New structured data type for transcript message content blocks with typed fields and query methods:
- `ContentBlock` with `block_type`, `text`, `tool_name`, `tool_input`, `tool_result_content`
- `last_assistant_used_tool(tool_name)` - check if assistant's last message used a specific tool
- `get_last_tool_use_in_message()` - get the last tool_use ContentBlock
- Real JSONL parsing replaces string-based matching

### Added - Shared Stop Hook Utilities

New `utils/stop_hook_helpers.py` module with `is_stop_hook_active()` and `get_transcript_reader()`, eliminating duplicated logic across stop handlers.

### Added - Source Guard CLAUDE.md Files

Added `CLAUDE.md` files to `src/` and `tests/` directories that prevent project-level agents from editing daemon source code, redirecting them to project-level handlers.

### Changed - Pipe Blocker Progressive Verbosity

First pipe block shows full verbose explanation with alternatives. Subsequent blocks show terse message with just the block reason and suggested command, reducing noise for experienced users.

### Changed - Stop Handlers Use Shared Utilities

`HedgingLanguageDetector` and `AutoContinueStopHandler` refactored to use shared `stop_hook_helpers` module, removing duplicated transcript reading and stop-hook-active detection logic.

### Fixed - AutoContinueStopHandler No Longer Overrides AskUserQuestion

Fixed bug where the auto-continue handler would override Claude's `AskUserQuestion` tool use, continuing automatically when the user was being asked a question. Handler now checks transcript for `AskUserQuestion` usage before triggering auto-continue.

### Fixed - Pipe Blocker Error Message Variable

Fixed pipe blocker using hardcoded `/tmp/output.txt` path in error messages instead of the `$TEMP_FILE` variable.
