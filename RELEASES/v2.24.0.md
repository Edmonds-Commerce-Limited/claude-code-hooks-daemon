# Release v2.24.0 - Upgrade Hang Fixes, Command Redirection CWD, Release Process Improvements

**Release Date:** 2026-03-18
**Type:** Minor Release

## Summary

Version 2.24.0 fixes two separate hangs in the upgrade/daemon start process that affected every project running the upgrade, fixes command redirection running from the wrong directory, and adds README stat tracking to the release process to prevent stale documentation.

## Highlights

- **Two upgrade hang fixes**: The upgrade script blocked on `read` in non-interactive mode, and daemon start hung callers using `$()` command substitution due to inherited stderr keeping the pipe open
- **Command redirection CWD fix**: Redirected commands (pipe_blocker, npm_command, gh_issue_comments) now run from the project root instead of `/`
- **Release process improvements**: README stats (handler count, test count, event types) are now tracked and verified during releases
- **Dogfooding mandate**: Formalised the rule that bugs found while using the daemon must be fixed immediately

## Changes

### Fixed

#### Daemon Start Hangs Callers Using `$()` Command Substitution

The double-fork daemonization in `cli.py cmd_start()` redirected stdout to `/dev/null` but intentionally kept stderr open for error output. When callers like `daemon_control.sh` captured output with `$("$python" ... start 2>&1)`, the grandchild daemon process held the pipe's write-end open via inherited stderr, causing the `$()` substitution to block forever.

**Fix**: Redirect both stdout and stderr to `/dev/null` in the grandchild process. The in-memory log system (MemoryLogHandler) already captures all errors — stderr is not needed for a properly daemonized background process.

**Files changed**: `src/claude_code_hooks_daemon/daemon/cli.py`, `tests/integration/test_plugin_daemon_integration.py`

#### Upgrade Script Hangs in Non-Interactive Mode

`scripts/upgrade_version.sh` used `read -r -p` for an upgrade guide confirmation prompt without checking whether stdin is a terminal. In non-interactive environments (Claude Code agents, CI pipelines), `read` blocks forever waiting for input that never arrives.

**Fix**: Added `[ ! -t 0 ]` detection to auto-skip the interactive prompt, with an informational message directing users to review upgrade guides after the upgrade completes.

**Files changed**: `scripts/upgrade_version.sh`

#### Command Redirection Runs in Project Root

`execute_and_save()` now accepts a `cwd` parameter, and all three callers (`gh_issue_comments`, `npm_command`, `pipe_blocker`) pass `ProjectContext.project_root()`. Previously, redirected commands ran from `/` (the daemon's cwd after daemonization with `os.chdir("/")`) causing them to fail.

Discovered via dogfooding when the pipe_blocker redirected a pytest command to a file but ran it from the wrong directory.

**Files changed**: `src/claude_code_hooks_daemon/core/command_redirection.py`, `src/claude_code_hooks_daemon/handlers/pre_tool_use/pipe_blocker.py`, `src/claude_code_hooks_daemon/handlers/pre_tool_use/npm_command.py`, `src/claude_code_hooks_daemon/handlers/pre_tool_use/gh_issue_comments.py`

#### README Stats Corrected

Updated README.md with accurate stats — handler count 48 to 68, event types 10 to 13, test count 6,255 to 7,344+. Clarified LLM-targeted sections (raw URLs at top) with collapsible `<details>` blocks labelled "LLM Quick Reference" to reduce confusion for human readers.

**Files changed**: `README.md`

### Added

#### README Stats Check in Release Process

The Opus documentation review step now explicitly validates that README.md stats are up to date — including the test count badge, test count in body text, handler count, and event type count. The release agent checklist and RELEASING.md documentation include an explicit step to update these stats as part of version updates.

**Files changed**: `.claude/agents/release-agent.md`, `CLAUDE/development/RELEASING.md`

#### Dogfooding Bug Fix Mandate

Added non-negotiable dogfooding rule to project CLAUDE.md — any bugs found while using the daemon's own handlers during development must be fixed immediately, not deferred.

**Files changed**: `CLAUDE.md`
