## Context

During plan 00080, the agent used dismissive language to deflect shellcheck warnings. The Stop-event-based `DismissiveLanguageDetectorHandler` failed to fire (Stop events are unreliable — fire once per session, not per turn). We need quality enforcement at a reliable inflection point.

The initial implementation (commit `37c5c3f` + uncommitted NitpickHandler) was architecturally wrong — it built a monolithic PreToolUse handler that internally called "checker strategies". The user corrected this: **nitpick is a pseudo-event that handlers attach to**, not an internal strategy dispatch.

## What Is a Pseudo-Event?

A **pseudo-event** is a synthetic event generated by the daemon, triggered by real events but with its own handler chain, dispatch, and configuration. Handlers register FOR pseudo-events the same way they register for PreToolUse or Stop — as independent Handler subclasses that can block, advise, or allow.

### Key Properties

1. **Multi-event triggers**: A pseudo-event hooks into real events via an array: `["pre_tool_use:1/5", "stop:1/1"]`
2. **Frequency control**: Notation `event_type:N/D` means "fire N times every D occurrences" (e.g., `1/5` = every 5th)
3. **Own config section**: Lives under `pseudo_events:` not `handlers:`
4. **Full handler chain**: Each pseudo-event handler is a Handler subclass with priority, terminal, matches/handle
5. **Shared infrastructure**: The pseudo-event fires a "setup" phase (e.g., transcript reading) ONCE, then all handlers receive the prepared data

### Dispatch Flow

```
Real Event (PreToolUse) arrives
  → DaemonController.process_event()
    → Route through real handler chain → real_result
    → PseudoEventDispatcher.check_triggers(event_type)
      → Counter check: should nitpick fire? (every Nth occurrence)
      → If yes:
        → Setup phase: read transcript, extract new assistant messages
        → Build pseudo_hook_input (original fields + assistant_messages)
        → Route through nitpick handler chain → pseudo_result
    → Merge real_result + pseudo_result → final_result
  → Return final_result
```

### Config Structure

```yaml
pseudo_events:
  nitpick:
    enabled: true
    triggers:
      - "pre_tool_use:1/5"   # every 5th PreToolUse
      - "stop:1/1"           # every Stop event
    handlers:
      dismissive_language:
        enabled: true
      hedging_language:
        enabled: true
```
