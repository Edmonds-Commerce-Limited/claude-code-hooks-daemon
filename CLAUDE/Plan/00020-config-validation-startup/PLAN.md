**Status**: Not Started

**Priority**: HIGH
**Type**: Feature
**GitHub Issue**: #13

## Overview

Implement comprehensive config validation at daemon startup with graceful fail-open error handling. Invalid config must not crash the daemon or allow silent operation without protection.

## Problem

Currently:
- No ConfigValidator.validate() call at daemon startup
- Forwarders silently fall back to defaults on config errors
- No "degraded mode" - daemon either starts or crashes
- Inconsistent error handling across code paths

## Solution

1. Run ConfigValidator.validate() at startup
2. Daemon starts in "degraded mode" on invalid config
3. Return HookResult.configuration_error() on every request when degraded
4. Standardize all error paths to use fail-open HookResult pattern
5. CLI shows degraded status with clear remediation steps
